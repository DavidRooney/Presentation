SVN to Git Migration

Approach & Aims

  • Make use of an already existing guide
  • Retain a complete history of the code
  • No need to go back to the SVN repo
  • Keep svn as a "just in case" back up
  • Better continuious integration using Git branches, Git Hooks & Git workflows

Challenges

  • Developers to move to Git without disruption
  • Ability to sync SVN to Git during above process
  • Retain a comprehesive code history
